# 双指针知识点大全

> 用两个指针遍历,把 O(n²) 优化到 O(n) 的核心套路

---

## 一、什么是双指针?

**核心**:用**两个指针**在数组/链表/字符串上遍历,代替暴力枚举。

**为什么快**:暴力枚举两个位置要 O(n²),双指针**有序地**移动,只走一遍 → **O(n)**。

## 二、双指针的三大类

```
1. 同向双指针(快慢指针)    2. 相向双指针(对撞指针)    3. 背向双指针
                           
→→                          ← →                        ← →
|__|__|__|__|__|            |__|__|__|__|__|           |__|__|__|__|
 ↑  ↑                       ↑              ↑                 ↑
慢  快                     左             右               中(向两边)
```

| 类型 | 起点 | 移动 | 典型题 |
|---|---|---|---|
| 同向(快慢) | 同一端 | 都向前 | 移除元素、链表环 |
| 相向(对撞) | 两端 | 向中间 | 两数之和、接雨水 |
| 背向 | 同一点 | 向两边 | 最长回文子串 |

## 三、同向双指针(快慢指针)

### 模板

```python
slow = 0
for fast in range(len(nums)):
    if 满足某种条件:
        nums[slow] = nums[fast]    # 把 fast 的值「搬」到 slow
        slow += 1
return slow    # slow 就是新数组的长度
```

### 经典例题:LC27 移除元素

```
nums = [3, 2, 2, 3],val = 3
输出:[2, 2],长度 2
```

```python
def removeElement(nums, val):
    slow = 0
    for fast in range(len(nums)):
        if nums[fast] != val:        # 满足「不是要删的」
            nums[slow] = nums[fast]
            slow += 1
    return slow
```

**走一遍**:
| 步 | fast | nums[fast] | 条件 | slow | 操作 |
|---|---|---|---|---|---|
| 1 | 0 | 3 | =val,跳过 | 0 | - |
| 2 | 1 | 2 | ≠val | 0 | nums[0]=2, slow=1 |
| 3 | 2 | 2 | ≠val | 1 | nums[1]=2, slow=2 |
| 4 | 3 | 3 | =val,跳过 | 2 | - |

返回 2,nums = [2, 2, 2, 3] ✓

### 同向指针的常见变体

- **LC26 删除重复项**:fast 跟 slow-1 比,不同就搬
- **LC283 移动零**:把 0 当 val,跟 LC27 一样
- **LC141 链表环**:fast 走两步,slow 走一步
- **LC876 链表中点**:fast 走两步,slow 走一步

## 四、相向双指针(对撞指针)

### 模板

```python
left, right = 0, len(nums) - 1
while left < right:
    if nums[left] + nums[right] == target:
        return [left, right]    # 找到了
    elif nums[left] + nums[right] < target:
        left += 1                # 和太小,左边要更大
    else:
        right -= 1               # 和太大,右边要更小
```

### 经典例题:LC1 两数之和(已排序版)

```
nums = [2, 7, 11, 15],target = 9
输出:[0, 1]
```

```python
def twoSum(nums, target):
    left, right = 0, len(nums) - 1
    while left < right:
        s = nums[left] + nums[right]
        if s == target:
            return [left, right]
        elif s < target:
            left += 1
        else:
            right -= 1
```

**走一遍**:
| 步 | left | right | sum | 9 | 动作 |
|---|---|---|---|---|---|
| 1 | 0 | 3 | 17 | > | right-=1 |
| 2 | 0 | 2 | 13 | > | right-=1 |
| 3 | 0 | 1 | 9 | = | 返回 [0, 1] ✓ |

**为什么这样对?**
- 数组已排序,和太大就缩小右边,和太小就扩大左边,**单调性保证能找到(或确定找不到)**

### 进阶:LC15 三数之和

LC1 的直接推广 — 固定一个数,在剩余数组里找两数之和 = 目标。

```python
def threeSum(nums):
    nums.sort()
    res = []
    for i in range(len(nums) - 2):
        if i > 0 and nums[i] == nums[i-1]:
            continue                # 跳过重复
        if nums[i] > 0:
            break                    # 剪枝
        left, right = i+1, len(nums)-1
        while left < right:
            s = nums[i] + nums[left] + nums[right]
            if s == 0:
                res.append([nums[i], nums[left], nums[right]])
                while left < right and nums[left] == nums[left+1]:
                    left += 1
                while left < right and nums[right] == nums[right-1]:
                    right -= 1
                left += 1
                right -= 1
            elif s < 0:
                left += 1
            else:
                right -= 1
    return res
```

**核心**:
- 外层循环定第一个数
- 内层双指针找后两个
- **跳过重复** 防重复答案

## 五、相向双指针经典题:LC42 接雨水

```
输入:height = [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]
输出:6
```

**核心思想**:用左右指针 + 左右最高,逐步逼近。

```python
def trap(height):
    if not height:
        return 0
    left, right = 0, len(height) - 1
    left_max = right_max = 0
    water = 0
    while left < right:
        if height[left] < height[right]:
            if height[left] >= left_max:
                left_max = height[left]
            else:
                water += left_max - height[left]
            left += 1
        else:
            if height[right] >= right_max:
                right_max = height[right]
            else:
                water += right_max - height[right]
            right -= 1
    return water
```

**原理**:
- 短板效应:能接多少水由**较矮**的那一侧决定
- 每次处理**较矮**的那边,看能不能接到水

## 六、滑动窗口(双指针的进阶)

### 什么是滑动窗口?

维护一个**区间** `[left, right]`,满足某个条件,**right 一直扩,left 缩到不能缩为止**。

```
[           ]  ← 窗口
left       right

→ 扩大:right += 1
→ 收缩:left += 1
```

### 模板(求最大窗口)

```python
left = 0
res = 0
window = {}        # 窗口里的状态(常用 Counter / set)
for right in range(len(s)):
    # 1. 把 s[right] 加进窗口
    window[s[right]] = window.get(s[right], 0) + 1
    
    # 2. 当窗口不满足条件时,左侧收缩
    while 窗口不满足条件:
        window[s[left]] -= 1
        if window[s[left]] == 0:
            del window[s[left]]
        left += 1
    
    # 3. 更新答案
    res = max(res, right - left + 1)

return res
```

### 模板(求最小窗口)

```python
left = 0
res = inf
window = {}
for right in range(len(s)):
    window[s[right]] = ...
    
    while 窗口满足条件:
        res = min(res, right - left + 1)
        window[s[left]] -= 1
        left += 1
return res
```

### 经典例题:LC3 无重复字符的最长子串

```
输入:s = "abcabcbb"
输出:3("abc" 最长)
```

```python
def lengthOfLongestSubstring(s):
    window = {}
    left = 0
    res = 0
    for right in range(len(s)):
        window[s[right]] = window.get(s[right], 0) + 1
        
        # 窗口里有重复了
        while window[s[right]] > 1:
            window[s[left]] -= 1
            left += 1
        
        res = max(res, right - left + 1)
    return res
```

**走一遍** "abcabcbb":
| right | s[right] | 窗口 | 动作 | res |
|---|---|---|---|---|
| 0 | a | {a:1} | 不收缩 | 1 |
| 1 | b | {a:1, b:1} | 不收缩 | 2 |
| 2 | c | {a:1, b:1, c:1} | 不收缩 | 3 |
| 3 | a | {a:2,...} | 收缩到 a:1 | 3 |
| ... | | | | |

## 七、链表双指针(快慢指针)

### LC141 链表是否有环

```python
def hasCycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:
            return True
    return False
```

**原理**:fast 每次 2 步,slow 每次 1 步,有环必相遇。

### LC876 链表的中间节点

```python
def middleNode(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow
```

**原理**:fast 走完时,slow 正好在中间。

## 九、必须背熟的小技巧

### 1. 什么时候用双指针?

- ✅ 数组/链表已**排序**
- ✅ 需要**配对**(两数之和)
- ✅ 需要**区间**(滑动窗口)
- ✅ 链表**环**或**中点**
- ❌ 完全无序 + 不需要配对 → 别硬用

### 2. 三数/四数之和的关键:跳过重复

```python
if i > 0 and nums[i] == nums[i-1]:
    continue    # 跳过重复,避免重复答案
```

### 3. 滑动窗口的两个时机

- **扩大窗口**:`right += 1`(总是前进)
- **缩小窗口**:`left += 1`(满足条件后才缩)

### 4. 链表双指针的两种步伐

| 问题 | fast | slow |
|---|---|---|
| 链表环 | 2 步 | 1 步 |
| 中点 | 2 步 | 1 步 |
| 倒数第 k 个 | 2 步 | 1 步(差 k) |

### 5. 字符串回文:对撞指针

```python
left, right = 0, len(s) - 1
while left < right:
    if s[left] != s[right]:
        return False
    left += 1
    right -= 1
return True
```

## 十、常见陷阱

1. **跳过重复漏掉**:`nums[i] == nums[i-1]`,不是 `nums[i+1]`
2. **left == right 时退出**:两个指针不能相遇
3. **滑动窗口忘记更新 res**:扩大和缩小都得考虑
4. **链表没判空**:`while fast and fast.next`
5. **三数之和漏掉剪枝**:外层太大就 break,内层和太大/小移动指针

## 十一、双指针速记表

| 模式 | 适用 | 模板关键字 |
|---|---|---|
| 同向去重/删除 | 数组、链表 | `slow, fast` |
| 相向两数之和 | 排序数组 | `left++, right--` |
| 滑动窗口最大 | 找最长满足 | `max(res, right-left+1)` |
| 滑动窗口最小 | 找最短满足 | `min(res, right-left+1)` |
| 链表环/中点 | 链表 | `fast.next.next` |
| 字符串回文 | 字符串 | 两端向中间 |
